# Kubernetes: подробный учебный материал — Gateway API: L4, gRPC и безопасная миграция

Полная теоретическая лекция для инженера, который начинает работать с Kubernetes. Термины на английском приведены в скобках там, где они используются в API или официальной документации.

## L4 и L7

TCPRoute направляет непрозрачный TCP-поток, UDPRoute — UDP, TLSRoute маршрутизирует TLS по SNI без termination при поддержке implementation. HTTPRoute/GRPCRoute работают на L7 и могут использовать правила протокола. Выбор уровня определяет, где заканчивается TLS, какие метрики доступны и можно ли выполнять header/path routing.

Изучайте ресурс через Kubernetes API, а не как отдельный YAML-файл. API server хранит желаемое состояние, а контроллеры асинхронно приводят кластер к нему. Поэтому момент успешного apply и момент готовности приложения различны.

Для любого объекта различайте spec и status. spec задаёт намерение, status и conditions сообщают результат работы контроллеров. Conditions с полями type, status, reason и message обычно точнее краткой колонки kubectl get.

События (Events) полезны как временная диагностическая подсказка. Их нельзя считать долговременным аудитом: сохраняйте важные факты в системе логирования и runbook. Всегда проверяйте ownerReferences, чтобы понять, какой контроллер создал объект.

Безопасная диагностика идёт сверху вниз: пользовательский результат, ресурс верхнего уровня, его потомки, node/runtime и внешняя зависимость. Массовое удаление Pod или выдача cluster-admin может скрыть причину, а не устранить её.

## Официальные источники для темы

Rollback должен быть подготовлен до cutover: предыдущий DNS/LB маршрут, сертификаты, сохранённый Ingress/values, критерий отмены и владелец решения. После переключения наблюдайте route conditions, TLS errors, 4xx/5xx, latency, connection count и результаты реальных клиентов.

Составьте таблицу поведения старого Ingress: class, hostname, path, TLS, annotations, auth, rate limit, source IP, timeout, canary, backend и внешние DNS/LB. Для каждой строки укажите стандарт Gateway API, implementation policy или отдельный компонент. Сначала поднимите параллельный listener/test hostname, затем выполните синтетические и прикладные проверки, после чего контролируемо переключайте DNS с учётом TTL и draining.

## Углубление: план миграции

Retries на edge не нейтральны: повтор POST или неидемпотентного RPC создаёт двойную операцию. Ограничивайте retry только случаями, которые безопасны и понятны приложению, а timeout согласуйте с client deadlines и backend.

TCPRoute не знает HTTP headers и не может выбирать path; TLSRoute использует SNI, но не расшифровывает поток; HTTPRoute/GRPCRoute видят L7 только после подходящего TLS-режима. Выбирайте самый низкий уровень, который решает задачу, чтобы не добавлять proxy и хранение сертификатов без причины. Для gRPC проверяйте HTTP/2, streaming, deadlines и response semantics.

## Домашняя практика

В учебном кластере создайте минимальный пример рассматриваемой сущности. До apply выполните server-side dry-run и diff. После применения сохраните YAML, describe, events и логи. Внесите одну намеренную ошибку и объясните, какой компонент и какая condition показывают её причину. Production-секреты и данные в упражнении не используйте.
